agent-tools 使用指南

统计 Claude Code 等 AI 编程助手的使用数据。自动采集、定时上报。

安装

从 Dashboard 右上角「下载客户端」获取安装包，下载后执行：

npm install -g agent-tools-cli-x.x.x.tgz

安装会自动：

  • 写入 Claude Code / CodeBuddy 的 hooks，开始采集
  • 配置服务器地址
  • （macOS/Windows）安装 guard 守护进程，防止 cc-switch 等工具覆盖配置

验证：

agent-tools --version
agent-tools status

配置用户标识 （macOS 用户必做）

macOS 用户必须配置：Mac 的 OS 登录名通常是个人化的（如 leon、perseus），跟你在 Dashboard 用的 SSO 身份（如 leon@corp.com）对不上，不配置会在 Dashboard 看不到自己的数据。

编辑 ~/.agent-tools/config.json，加一行 username，值就是你登录 Dashboard 的那个 SSO 名：

{
  "server": { "url": "https://..." },
  "username": "leon@corp.com"
}

不知道自己的 SSO 名？打开 Dashboard 看右上角，登录后的那个名字就是。

下次 hook 触发后，新事件就会用这个标识上报。已上报的旧数据不动。升级客户端时该字段会被保留，不需要重配。

让 Claude Code 帮你一键配置： 
请帮我编辑 ~/.agent-tools/config.json，加一个 username 字段，值是 [我的SSO名]。
注意：保留文件里现有的其他字段（server.url、sync 等）不要动。

遇到问题？让 Claude Code 帮你排查

推荐：把下面整段「排查脚本」复制粘贴给 Claude Code，让它自己跑命令、读输出、对照下面的「常见症状表」给出诊断和修复建议。

给 Claude Code 的排查脚本

我装了 agent-tools 但遇到问题（[在这里描述：无数据 / 没有 hooks 触发 / 装不上 / ……]）。
请按下面的步骤帮我排查：

1. 跑下面这些诊断命令，把每一条的输出都给我看：

# 客户端版本
agent-tools --version

# 客户端 + 已检测到的 agent + hooks 状态
agent-tools status

# 客户端配置（脱敏看一下 server.url、username）
cat ~/.agent-tools/config.json

# 本地事件队列状态（synced=0 表示未上报）
sqlite3 ~/.agent-tools/data/local.db \
  "SELECT synced, COUNT(*) FROM local_events GROUP BY synced;"

# 最近一次同步的实际请求和响应（试一次，看输出）
DEBUG_AGENT_TOOLS=1 agent-tools sync 2>&1

# Claude Code hooks 是否注入
cat ~/.claude/settings.json 2>/dev/null | python3 -m json.tool | grep -A2 "hooks\|universal-hook" | head -40

# 自动更新 / 安装日志
cat ~/.agent-tools/data/update-log.json 2>/dev/null

# 服务器是否可达
SERVER_URL=$(cat ~/.agent-tools/config.json | python3 -c "import sys,json; print(json.load(sys.stdin)['server']['url'])")
curl -sS -o /dev/null -w "HTTP %{http_code}  time %{time_total}s\n" "$SERVER_URL/api/v1/health"

2. 把上面输出对照下面的「常见症状」表给我诊断（症状 → 根因 → 修复命令），
   然后给我一条具体的修复命令。不要让我手填路径。

3. 如果修复需要重启 Claude Code 才能生效，明确告诉我。

强制覆盖。仍不行，重启 Claude Code 后再观察。
切换 cc-switch 供应商后 hooks 消失 agent-tools status 显示 cc-switch installed=true 但 protected=false；
或 settings.json 里没 hooks 但之前有 
agent-tools setup --force
会把 hooks 写进 cc-switch 的「通用配置」并对所有 claude 供应商启用 commonConfigEnabled。 新增供应商后需要再跑一次。
命令找不到 agent-tools: command not found which agent-tools 空；npm list -g agent-tools 空 全局安装路径不在 PATH。
npm config get prefix
# 把上面输出的 prefix/bin 加进 PATH
或重新装：npm install -g <tgz>
自动更新失败 / 卡在旧版本 ~/.agent-tools/data/update-log.json 里有 status: "failed" 条目 看 error 字段：
• EACCES → 全局 npm 没写权限，需要 sudo 重装
• ENOTFOUND → 服务器不可达
• 其他 → 手动重装最新 tgz
服务器返回 401 / 403 DEBUG_AGENT_TOOLS=1 agent-tools sync 看到 401/403 server URL 错了或者打到了 SSO 登录页。检查 config.json 里 server.url 是不是 API 根（不带路径）， 再 curl <server>/api/v1/health 看 200。
本地事件持续堆积（unsynced 越来越多） SELECT COUNT(*) FROM local_events WHERE synced=0; 持续增长 服务端持续报 errors。先按上面"sync 显示成功但服务端无数据"流程排查。 确认根因修了之后，本地事件会自动重发上去（event_id 唯一去重，不会重复入库）。
guard 占 CPU / 不停拉起 macOS launchd 或 Windows 任务计划里反复 spawn 一个 node 进程 
agent-tools guard uninstall
agent-tools guard status   # 确认已移除
然后再 agent-tools setup --force 重建，或保持禁用：在 config.json 加 "guard": {"enabled": false}
