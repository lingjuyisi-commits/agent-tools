1. Dashboard "我的统计" 没有数据

Step 1：确认客户端已安装

agent-tools --version

如果报 "command not found"，说明还没装。装完之后 postinstall 会自动注入钩子。

Step 2：确认钩子已注入到 Claude Code

打开 ~/.claude/settings.json，在 hooks 字段里找 universal-hook.js 这个关键字。

  • 能找到：钩子正常，Claude Code 下次调工具就会触发事件，进入 Step 3。
  • 找不到：钩子被抹了，执行： 
    agent-tools setup --force
    重新注入。

Step 3：确认本地已产生事件

agent-tools stats

看数字是不是 0。是 0 就在 Claude Code 里随便跑一两条命令（任何工具调用都算），再重试。

Step 4：手动触发一次同步

agent-tools sync

同步后刷新 Dashboard。如果报网络错误或 401/403，检查 ~/.agent-tools/config.json 里的 server.url。

2. 每次用 cc-switch 切 key 后数据就停止

原因：cc-switch 切换供应商时会整块重写 ~/.claude/settings.json。我们的钩子如果不在 cc-switch 的"通用配置"里，就会被抹掉。

方案 A（推荐）：把钩子加到 cc-switch 通用配置

一次配置，长期有效。cc-switch 每次切供应商时会把通用配置作为 runtime overlay 叠加到 live settings.json 上，钩子就不会丢。

前置：先执行 agent-tools setup --force 确认钩子已注入到 ~/.claude/settings.json。 下面的"从当前供应商提取"会从这份 live 文件读取，所以必须先有钩子再提取。
  1. 升级 cc-switch 到 v3.12.0 或更高版本（v3.11.0 有已知 bug，要跳过）。
  2. 打开 cc-switch，找到当前正在使用的供应商卡片，hover 露出操作按钮，点 "编辑"。
  3. 在编辑面板里切到 "通用配置面板"。
  4. 点 "从当前供应商提取" —— cc-switch 会把 settings.json 里除了 API Key / endpoint 之外的所有字段拉进来，包括我们注入的 hooks。
  5. 确认提取出来的 JSON 里每个事件下面都出现了 universal-hook.js 的 command。
  6. 点 "保存"。通用配置立即生效，不需要重启 cc-switch 或 CLI。

之后：

  • 任何时候切 key → 通用配置作为 overlay 叠到 settings.json，钩子一直在。
  • 创建新供应商时，"写入通用配置" 复选框默认勾上 —— 保持勾选就好，钩子会跟着新 provider 一起生效。

方案 B：让 guard 守护进程兜底

没方便配通用配置？guard 会监听 ~/.claude/settings.json，钩子一被抹就重新注入。

agent-tools guard install
agent-tools guard status

查看 guard 日志：

cat ~/.agent-tools/data/guard-log.json

看到 "event": "healed" 的条目就说明它在工作。

3. 我是管理员但看不到全员数据

  1. 确认登录身份：Dashboard 右上角显示的 login 要和 allowed_users 表里的一致。
  2. 确认 role：在 "管理员管理" Tab 里找到自己，确认是 admin。
  3. 访问 /auth/session，返回里 isAdmin 必须是 true。

如果 isAdmin: false，联系已有的管理员把你的 login 加进白名单。

4. 我确认产过事件但 "我的统计" 仍然是空的

agent-tools 假设 SSO 登录的 login 和 CLI 上报的 username 一致。如果你的 OS 用户名（比如 alice.li）和 SSO 身份（比如 alice_li）不一致，就匹配不上。

自查步骤：

# 看 CLI 上报用了啥 username
node -e "const os=require('os'); console.log(os.userInfo().username)"

# 对比 Dashboard 右上角的 SSO login

两者不一致时联系管理员协调 username 对齐（或申请统一改成 SSO login）。
