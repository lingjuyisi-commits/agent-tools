1. 客户端安装失败

判定标准：

agent-tools --version

如果报 command not found、或者直接执行报错，都属于这一类。

1.1 command not found

  • 确认 Node.js 和 npm 已安装： 
    node --version
npm --version
    没装就先装 Node ≥ 20。
  • 确认 npm 全局 bin 在 PATH 里： 
    npm config get prefix
# 检查 $PATH / %PATH% 里有没有上面这个路径下的 bin / 子目录
    Windows 常见位置是 %APPDATA%\npm；macOS/Linux 常见是 /usr/local/bin 或 ~/.npm-global/bin。
  • 如果之前装过但被卸了，直接重装就行： 
    npm install -g <your-agent-tools-tgz-url-or-registry>

1.2 npm install 报权限错误

  • macOS / Linux 报 EACCES：不要用 sudo 硬装。改成用户级 npm prefix： 
    mkdir ~/.npm-global
npm config set prefix ~/.npm-global
export PATH=~/.npm-global/bin:$PATH   # 加到 ~/.bashrc / ~/.zshrc
npm install -g <tgz>
  • Windows：一般装到 %APPDATA%\npm 不需要管理员权限；如果被企业策略拦了，联系 IT 放行 npm.cmd 执行。

1.3 postinstall 报错 / 中途失败

npm 安装成功但 postinstall 报错，会看到类似 [agent-tools] guard 配置失败（已忽略） 的行。常见原因：

  • better-sqlite3 原生编译失败：prebuilt 下载失败或不匹配。装带 full 后缀的离线包，里面已经预编译好： 
    npm install -g agent-tools-cli-full-<platform>-<version>.tgz
  • 企业 EDR / 杀毒拦了 schtasks（Windows）或 launchctl（macOS）：CLI 本身能用，只是 guard 装不上。不是致命问题，手动跑一次检查： 
    agent-tools guard status
    如果企业安全策略放不开，又想禁止自动装 guard，改配置： 
    # ~/.agent-tools/config.json
{ "guard": { "enabled": false }, ... }
    然后下次自升级 postinstall 会跑 guard.uninstall() 清掉残留。
  • 网络到不了 server：default-config.json 里 server.url 为空，postinstall 只完成本地配置文件但不会自动注入 hook。手动补： 
    agent-tools init --server <your server url>
agent-tools setup --force

1.4 装完了之后呢

agent-tools --version 正常输出版本号就算装好了。然后在 Claude Code 里随便执行一条命令，等几分钟，如果 Dashboard 还是没数据 → 进第 2 节。

2. 安装成功，但 Dashboard 没有我的数据

四步自检

Step 1：钩子是否在 ~/.claude/settings.json 里

打开 ~/.claude/settings.json，在 hooks 字段里搜 universal-hook.js。

  • 能找到：钩子健在，继续 Step 2。
  • 找不到 / 钩子字段整块没了：被外部工具（大概率 cc-switch）抹了。先手动注入一次，再看 第 2.2 节根治。 
    agent-tools setup --force

Step 2：本地是否有事件

agent-tools stats

如果所有指标都是 0：Claude Code 没触发过钩子。去 Claude Code 里执行一条真实的工具调用（编辑文件、运行命令都行），回头再查。

如果有数字：本地采集正常，问题在同步环节，继续 Step 3。

Step 3：手动同步一次

agent-tools sync
  • 报 网络错误：检查 ~/.agent-tools/config.json 里的 server.url，以及你机器能不能访问那个地址。
  • 报 401 / 403：检查 server 端鉴权配置，或者是否需要先登录。
  • 没报错但 Dashboard 还是空的：刷新 Dashboard，看右上角登录身份和你 CLI 机器的 OS 用户名是否一致。

Step 4：看 CLI 上报日志

cat ~/.agent-tools/data/update-log.json
cat ~/.agent-tools/data/guard-log.json   # 如果启用了 guard

失败的同步 / heal 行为会在这里留痕。

2.2 被 cc-switch 抹钩子的专项处理

症状：刚装好时数据正常，用了 cc-switch 切 key 之后就突然不再上报。

原因：cc-switch 切供应商时会用存储里那个 provider 的 snapshot + 通用配置 overlay 覆盖 ~/.claude/settings.json。我们的钩子如果不在 cc-switch 的"通用配置"里，overlay 时就没了。

方案 A（推荐）：把钩子加到 cc-switch 通用配置

一次配置，长期有效。cc-switch 每次切 provider 会把通用配置作为 runtime overlay 叠到 live settings.json 上，钩子就不会丢。

前置：先执行 agent-tools setup --force，确认钩子已回到 ~/.claude/settings.json。下面的"从当前供应商提取"是从这份 live 文件读的，所以钩子必须先在。
  1. 升级 cc-switch 到 v3.12.0 或更高版本（v3.11.0 有已知 bug，跳过）。
  2. 打开 cc-switch，找到当前正在使用的供应商卡片，hover 露出操作按钮，点 "编辑"。
  3. 在编辑面板里切到 "通用配置面板"。
  4. 点 "从当前供应商提取" —— cc-switch 会把 settings.json 里除 API Key / endpoint 之外的所有字段拉进来，包括我们注入的 hooks。
  5. 确认提取出来的 JSON 里每个事件下面都有 universal-hook.js 的 command。
  6. 点 "保存"。通用配置立即生效，不需要重启 cc-switch 或 CLI。

之后：

  • 任何时候切 key（主界面 "启用" 按钮或系统托盘右键快捷切都一样）→ 通用配置 overlay，钩子一直在。
  • 新建供应商时，"写入通用配置" 复选框默认勾上 —— 切勿取消勾选。取消的话那个 provider 切过去时不叠 overlay，钩子又会丢。
⚠️ 通用配置是快照，不是活链接
"从当前供应商提取"只拍一次快照。以后 agent-tools 升级、新增钩子事件时，settings.json 会多出新 hook，但通用配置还是老的快照 —— 下次切 provider 时，新 hook 会被 overlay 覆盖掉。

应对：agent-tools 每次升级后，重复一遍"从当前供应商提取 → 保存"。或者开着 guard 兜底（方案 B），guard 会在钩子被抹的瞬间自动补回来。

误操作救回：cc-switch 会自动把最近 10 份配置快照存到 ~/.cc-switch/backups/，搞坏了通用配置可以从里面恢复。

方案 B：让 guard 守护进程兜底

不方便配通用配置？guard 会监听 ~/.claude/settings.json，钩子一被抹就自动重新注入。

agent-tools guard install
agent-tools guard status

查看 guard 日志（有 "event": "healed" 的条目就说明在工作）：

cat ~/.agent-tools/data/guard-log.json
